# プロダクト要求定義書

## プロダクト概要

### プロダクト名

個人ポートフォリオサイト

### ビジョン

エンジニアとしての技術力、実績、人となりを効果的に伝える、モダンで洗練されたポートフォリオサイト。

### 目的

- 自身のスキルセットと経験を明確に示す
- 採用担当者や協業希望者に対して専門性をアピール
- アウトプット（プロジェクト、記事など）の発信基盤とする

## ターゲットユーザー

### プライマリ

- 採用担当者・リクルーター
- 技術職の面接官

### セカンダリ

- 協業を検討するエンジニア
- 技術コミュニティのメンバー

## 主要機能要求

### 必須機能（MVP）

1. **ヒーローセクション**
   - 名前・肩書きの表示
   - ファーストビューでの印象付け

2. **スキルスタック表示**
   - 言語・フレームワーク・ツールの一覧
   - 各スキルの使用経験・習熟度
   - アイコンによる視覚的表現

3. **アウトプットセクション**
   - プロジェクト・制作物の紹介
   - サムネイル画像とリンク

4. **お問い合わせフォーム**
   - 名前、メールアドレス、メッセージ入力
   - Formspree による送信機能

5. **レスポンシブデザイン**
   - デスクトップ・タブレット・スマートフォン対応
   - ハンバーガーメニュー（モバイル）

### 今後の拡張機能

- [ ] ダークモード切り替え
- [ ] Lapras APIとの連携

## 非機能要求

### セキュリティ

- HTTPS 必須
- XSS、CSRF対策
- 個人情報の適切な取り扱い

### 保守性

- コンポーネント化されたコード
- TypeScript による型安全性
- テストカバレッジ: 70%以上

## 制約事項

### 技術制約

- GitHub Pages でのホスティング（静的サイト）
- サーバーサイド処理は外部サービス利用（Formspree等）

### リソース制約

- 個人開発（1名）
- 予算: 最小限（無料プランの活用）

### スケジュール

- MVP リリース: 完了
- 継続的な改善: 随時
