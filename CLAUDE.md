# CLAUDE.md

このファイルは、Claude Code (claude.ai/code) がこのリポジトリで作業する際のマスタードキュメントです。

## ドキュメント管理ポリシー

### 永続的ドキュメント（このファイルと関連ドキュメント）

プロジェクト全体で参照される基準ドキュメント。プロジェクトが存続する限り継続的に更新されます。

- **CLAUDE.md**（本ファイル）: 開発ルール、ドキュメント管理方針
- **[プロダクト要求定義書](.claude/docs/product-requirements.md)**: プロダクトのビジョン、目的、要求事項
- **[技術仕様書](.claude/docs/architecture.md)**: アーキテクチャ、技術選定、UI/UX設計、API設計
- **[リポジトリ構造定義書](.claude/docs/repository-structure.md)**: ディレクトリ構造、ファイル命名規則

### ステアリングドキュメント（作業単位）

各タスク・機能開発ごとに作成される作業指示書。タスク完了後はアーカイブされます。

- **配置場所**: `.claude/steering/active/YYYYMMDD-{task-title}/`
- **構成要素**:
  - 作業の要求内容（機能説明、ユーザーストーリー、受け入れ条件、制約事項）
  - 変更内容の設計（変更コンポーネント、実装アプローチ、データ構造変更、影響範囲分析）
  - タスクリスト
- **アーカイブ**: 完了後は `.claude/steering/archive/` に移動

### ドキュメント更新フロー

1. 新しいタスク開始時: ステアリングドキュメントを作成
2. 作業中: ステアリングドキュメントに従って開発
3. 重要な変更: 永続的ドキュメントを更新
4. タスク完了: ステアリングドキュメントをアーカイブ

## ドキュメント作成のルール

### メンテナンス負荷を抑える
- **ソースコードを読めばわかることは書かない**
  - 例: 関数の実装詳細、具体的なコード例
  - 理由: コードとドキュメントの乖離が発生する

- **他のドキュメントと重複しない**
  - 同じ情報を複数箇所に書かない
  - 必要な場合はリンクで参照する
  - 例: ビルドコマンドは `CONTRIBUTING.md` に記載、`CLAUDE.md` からはリンク

- **自動取得可能な情報は書かない**
  - 例: ディレクトリツリー（`tree` や `ls` で取得可能）
  - 例: 依存パッケージ一覧（`package.json` で確認可能）

### 書くべき内容
- **方針・ルール**: なぜそうするのか（Why）、どうあるべきか（What）
- **設計思想**: 技術選定の理由、アーキテクチャの方針
- **ワークフロー**: 開発の進め方、ドキュメント更新フロー

### 図表の扱い
- 必要最小限に留める
- 設計変更時は図表も同時に更新
- コードと図表の乖離を防ぐ

## 注意事項

- ドキュメントの作成更新は段階的に行い、各段階で承認を得る
- 永続的ドキュメントと作業単位のドキュメントを混同しない
- 共通のデザインシステム（Tailwind CSS）を使用して統一感を保つ
